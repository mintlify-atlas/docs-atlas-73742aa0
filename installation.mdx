# Installation

Goanna can be deployed using Docker (recommended), Docker Compose, or built from source for development. This guide covers all installation methods.

## Docker Installation (Recommended)

The easiest way to run Goanna is using the pre-built Docker image. This single container includes both the web UI and the API.

### Basic Docker Run

```bash
docker run -d \
  --name goanna \
  -p 9044:9044 \
  -e GOANNA_WEB_HOST=0.0.0.0 \
  -e GOANNA_WEB_PORT=9044 \
  -e GOANNA_WEB_INTERNAL_PORT=9045 \
  -e GOANNA_API_ADDR=:8080 \
  -e GOANNA_API_DSN='file:/app/data/goanna.db?_fk=1' \
  -e GOANNA_MAX_RESPONSE_BODY_BYTES=25165824 \
  -v ./data:/app/data \
  ghcr.io/simbesh/goanna:latest
```

### Build Your Own Image

If you prefer to build the image yourself:

```bash
# Clone the repository
git clone https://github.com/simbesh/goanna.git
cd goanna

# Build the image
docker build -t goanna:latest --build-arg VITE_API_BASE_URL=/ .

# Run the container
docker run -d \
  --name goanna \
  -p 9044:9044 \
  -v ./data:/app/data \
  goanna:latest
```

<Note>
  `VITE_API_BASE_URL` is a build-time variable. Use `/` to route browser API calls through the web gateway in the same container.
</Note>

## Docker Compose Installation

For production deployments with better orchestration, use Docker Compose.

### Create docker-compose.yml

Create a `docker-compose.yml` file:

```yaml
services:
  goanna:
    image: ghcr.io/simbesh/goanna:latest
    # Optional: build locally instead of pulling
    # build:
    #   context: .
    #   dockerfile: Dockerfile
    #   args:
    #     VITE_API_BASE_URL: /
    environment:
      GOANNA_WEB_HOST: 0.0.0.0
      GOANNA_WEB_PORT: '9044'
      GOANNA_WEB_INTERNAL_PORT: '9045'
      GOANNA_API_ADDR: ':8080'
      GOANNA_API_DSN: file:/app/data/goanna.db?_fk=1
      GOANNA_MAX_RESPONSE_BODY_BYTES: '25165824'
    ports:
      - '9044:9044'
      # Optional: expose API publicly if needed
      # - '8080:8080'
    volumes:
      - ./data:/app/data
    restart: unless-stopped
```

### Start with Docker Compose

<Steps>
  <Step title="Start the services">
    ```bash
    docker compose up -d
    ```
  </Step>
  
  <Step title="View logs">
    ```bash
    docker compose logs -f goanna
    ```
  </Step>
  
  <Step title="Stop the services">
    ```bash
    docker compose down
    ```
  </Step>
</Steps>

## From Source Installation

For development or customization, you can run Goanna from source.

### Prerequisites

- Bun 1.3+
- Go 1.24+
- Git

### Installation Steps

<Steps>
  <Step title="Clone the Repository">
    ```bash
    git clone https://github.com/simbesh/goanna.git
    cd goanna
    ```
  </Step>
  
  <Step title="Bootstrap the Project">
    Install dependencies and generate code:
    
    ```bash
    bun run bootstrap
    ```
    
    This command:
    - Installs all npm dependencies
    - Generates Ent ORM code
    - Generates OpenAPI types
    - Generates TypeScript API client
  </Step>
  
  <Step title="Run the Backend">
    In one terminal:
    
    ```bash
    bun run dev:api
    ```
    
    The API will start on `http://localhost:8080`
  </Step>
  
  <Step title="Run the Frontend">
    In another terminal:
    
    ```bash
    bun run dev:web
    ```
    
    The web UI will start on `http://localhost:3000`
  </Step>
</Steps>

### Common Development Commands

```bash
# Frontend dev server
bun run dev:web

# Backend dev server
bun run dev:api

# Regenerate Ent ORM code
bun run gen:ent

# Regenerate OpenAPI code
bun run gen:api

# Build production web bundle
bun run build:web
```

## Environment Variables

Goanna uses environment variables for configuration. All variables are optional with sensible defaults.

### Web Configuration

<ParamField path="GOANNA_WEB_HOST" type="string" default="0.0.0.0">
  Host address for the web server
</ParamField>

<ParamField path="GOANNA_WEB_PORT" type="string" default="9044">
  External port for the web server
</ParamField>

<ParamField path="GOANNA_WEB_INTERNAL_PORT" type="string" default="9045">
  Internal port for the web server
</ParamField>

### API Configuration

<ParamField path="GOANNA_API_ADDR" type="string" default=":8080">
  Address and port for the API server
</ParamField>

<ParamField path="GOANNA_API_DSN" type="string" default="file:/app/data/goanna.db?_fk=1">
  SQLite database connection string. Format: `file:/path/to/db.db?_fk=1`
</ParamField>

<ParamField path="GOANNA_API_INTERNAL_URL" type="string" default="http://127.0.0.1:8080">
  Internal URL for API communication (used by web gateway)
</ParamField>

<ParamField path="GOANNA_MAX_RESPONSE_BODY_BYTES" type="integer" default="25165824">
  Maximum API response body size for monitor checks and selector payload caching (default: 24 MB)
</ParamField>

### Build-time Variables

<ParamField path="VITE_API_BASE_URL" type="string" default="/">
  Base URL for API calls from the browser. Use `/` to route through the web gateway in the same container.
</ParamField>

## Port Configuration

By default, Goanna exposes:

- **Port 9044**: Web UI (public)
- **Port 8080**: API (internal only)

The API is proxied through the web server at `/v1/*` and `/healthz` endpoints.

### Exposing the API Publicly

If you need direct API access:

<CodeGroup>
```bash Docker
docker run -d \
  --name goanna \
  -p 9044:9044 \
  -p 8080:8080 \
  -v ./data:/app/data \
  ghcr.io/simbesh/goanna:latest
```

```yaml Docker Compose
services:
  goanna:
    image: ghcr.io/simbesh/goanna:latest
    ports:
      - '9044:9044'
      - '8080:8080'  # Expose API publicly
    volumes:
      - ./data:/app/data
```
</CodeGroup>

## Data Persistence

### Database Location

Goanna uses SQLite for data storage. By default, the database is located at:

- **Docker**: `/app/data/goanna.db` (mount a volume to persist)
- **From Source**: Location specified in `GOANNA_API_DSN`

### Backup and Restore

<Steps>
  <Step title="Backup">
    ```bash
    # Stop the container
    docker stop goanna
    
    # Copy the database
    cp ./data/goanna.db ./backup/goanna-$(date +%Y%m%d).db
    
    # Restart the container
    docker start goanna
    ```
  </Step>
  
  <Step title="Restore">
    ```bash
    # Stop the container
    docker stop goanna
    
    # Restore the database
    cp ./backup/goanna-20260302.db ./data/goanna.db
    
    # Restart the container
    docker start goanna
    ```
  </Step>
</Steps>

Verify that Goanna is running:

```bash
curl http://localhost:9044/healthz
```

Expected response:
```json
{
  "status": "ok"
}
```

## Upgrading

### Docker

<Steps>
  <Step title="Pull the latest image">
    ```bash
    docker pull ghcr.io/simbesh/goanna:latest
    ```
  </Step>
  
  <Step title="Stop and remove the old container">
    ```bash
    docker stop goanna
    docker rm goanna
    ```
  </Step>
  
  <Step title="Start a new container">
    ```bash
    docker run -d \
      --name goanna \
      -p 9044:9044 \
      -v ./data:/app/data \
      ghcr.io/simbesh/goanna:latest
    ```
    
    <Note>
      Your data will be preserved as long as you mount the same volume.
    </Note>
  </Step>
</Steps>

### Docker Compose

```bash
docker compose pull
docker compose up -d
```

### From Source

```bash
git pull origin main
bun run bootstrap
```

## Troubleshooting

<AccordionGroup>
  <Accordion title="Port already in use">
    If port 9044 is already in use, change the port mapping:
    
    ```bash
    docker run -d \
      --name goanna \
      -p 8888:9044 \
      -v ./data:/app/data \
      ghcr.io/simbesh/goanna:latest
    ```
    
    Access Goanna at `http://localhost:8888`
  </Accordion>
  
  <Accordion title="Permission denied on data directory">
    Ensure the data directory is writable:
    
    ```bash
    mkdir -p ./data
    chmod 755 ./data
    ```
  </Accordion>
  
  <Accordion title="Database locked errors">
    This usually means multiple instances are trying to access the same database:
    
    1. Ensure only one Goanna instance is running
    2. Stop all containers: `docker stop $(docker ps -q --filter name=goanna)`
    3. Start a single instance
  </Accordion>
  
  <Accordion title="Build fails from source">
    Verify your environment:
    
    ```bash
    # Check Bun version
    bun --version  # Should be 1.3+
    
    # Check Go version
    go version     # Should be 1.24+
    ```
    
    Clean and rebuild:
    ```bash
    rm -rf node_modules
    bun install
    bun run bootstrap
    ```
  </Accordion>
</AccordionGroup>
