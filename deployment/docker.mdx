Goanna ships with a production-ready Dockerfile that packages both the web frontend and API backend into a single container image. This guide covers building and running Goanna with Docker.

## Overview

The Docker image includes:
- Web frontend (TanStack Start/Nitro) on port 9044
- API backend (Go + SQLite) internally on port 8080
- Built-in API gateway that proxies `/v1/*` and `/healthz` through the web port

By default, only the web port (9044) is exposed publicly. The API runs internally and is accessed through the web gateway.

## Building the Image

<Steps>

<Step title="Build the image">

The `VITE_API_BASE_URL` build argument configures how the web app routes API calls. Use `/` to route through the same container's gateway:

```bash
docker build -t goanna:latest --build-arg VITE_API_BASE_URL=/ .
```

<Note>
The `VITE_API_BASE_URL` is a **build-time** variable that gets embedded into the web bundle during compilation.
</Note>

</Step>

<Step title="Verify the build">

Check that the image was created successfully:

```bash
docker images | grep goanna
```

</Step>

</Steps>

## Running with docker run

<Steps>

<Step title="Create a data directory">

Goanna stores its SQLite database in a persistent volume. Create a local directory for data:

```bash
mkdir -p ./data
```

</Step>

<Step title="Run the container">

Start Goanna with all default environment variables:

```bash
docker run -d \
  --name goanna \
  -p 9044:9044 \
  -e GOANNA_WEB_HOST=0.0.0.0 \
  -e GOANNA_WEB_PORT=9044 \
  -e GOANNA_WEB_INTERNAL_PORT=9045 \
  -e GOANNA_API_ADDR=:8080 \
  -e GOANNA_API_DSN='file:/app/data/goanna.db?_fk=1' \
  -e GOANNA_MAX_RESPONSE_BODY_BYTES=25165824 \
  -v ./data:/app/data \
  goanna:latest
```

</Step>

<Step title="Access the application">

Open your browser and navigate to:

```
http://localhost:9044
```

</Step>

</Steps>

## Port Mapping

Goanna uses two ports internally:

| Port | Service | Exposed by Default |
|------|---------|-------------------|
| 9044 | Web frontend + API gateway | Yes |
| 8080 | API backend (internal) | No |

### Exposing the API Port

If you need direct access to the API (bypassing the gateway), add the API port mapping:

<CodeGroup>

```bash docker run
docker run -d \
  --name goanna \
  -p 9044:9044 \
  -p 8080:8080 \
  -v ./data:/app/data \
  goanna:latest
```

```yaml docker-compose.yml
services:
  goanna:
    image: goanna:latest
    ports:
      - '9044:9044'
      - '8080:8080'  # Expose API publicly
    volumes:
      - ./data:/app/data
```

</CodeGroup>

## Volume Management

Goanna requires a persistent volume to store the SQLite database:

```bash
-v ./data:/app/data
```

This mounts a local `./data` directory to `/app/data` inside the container, where the database file `goanna.db` is stored.

<Note>
Without a volume mount, all data will be lost when the container is removed.
</Note>

### Alternative Volume Options

<CodeGroup>

```bash Named Volume
# Create a named volume
docker volume create goanna-data

# Use it in docker run
docker run -d \
  --name goanna \
  -p 9044:9044 \
  -v goanna-data:/app/data \
  goanna:latest
```

```bash Absolute Path
docker run -d \
  --name goanna \
  -p 9044:9044 \
  -v /path/to/persistent/data:/app/data \
  goanna:latest
```

</CodeGroup>

## Dockerfile Structure

The production Dockerfile uses a multi-stage build:

<Steps>

<Step title="Web Builder Stage">

Builds the web frontend using Bun:

```dockerfile
FROM oven/bun:1.3 AS web-builder
WORKDIR /src

# Install dependencies
COPY package.json bun.lock ./
RUN bun install --frozen-lockfile

# Build production bundle
ARG VITE_API_BASE_URL=/
ENV VITE_API_BASE_URL=${VITE_API_BASE_URL}
RUN bun run --cwd apps/web build
```

</Step>

<Step title="API Builder Stage">

Compiles the Go API binary:

```dockerfile
FROM golang:1.24-bookworm AS api-builder
WORKDIR /src/apps/api

# Install C toolchain for go-sqlite3
RUN apt-get update && apt-get install -y build-essential ca-certificates

# Build binary
RUN go build -o /out/goanna-api ./cmd/server
```

</Step>

<Step title="Runtime Stage">

Combines both services in a minimal runtime image:

```dockerfile
FROM node:22-bookworm-slim AS runtime
WORKDIR /app

# Copy binaries and built assets
COPY --from=api-builder /out/goanna-api /app/bin/goanna-api
COPY --from=web-builder /src/apps/web/.output /app/web/.output

# Expose ports and define volume
EXPOSE 9044 8080
VOLUME ["/app/data"]
```

</Step>

</Steps>
